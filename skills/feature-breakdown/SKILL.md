---
name: feature-breakdown
description: Turn a vague feature request into an execution plan with boundaries, risks, and incremental steps.
tags: planning, implementation
---
When breaking down a feature:
1. Separate user-facing behavior from internal implementation choices.
2. Identify the minimal working version before proposing stretch additions.
3. List affected modules, data flow, and tool or model dependencies.
4. Surface ambiguities early instead of hiding them inside implementation.
5. Produce a plan that can be executed incrementally, verified at each step, and revised if assumptions change.
